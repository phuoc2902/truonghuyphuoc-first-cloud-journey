**Trong trang này** bạn sẽ cần giới thiệu worklog của bạn **như thế nào**? Bạn hoàn thành chương trình trong vòng **bao nhiêu tuần**? Bạn **đã làm gì** trong các tuần đó?


Thông thường và cũng là tiêu chuẩn, một worklog được thực hiện trong khoảng 3 tháng (trong suốt thời gian thực tập) với nội dung các tuần như sau:

**Tuần 1:** [Khởi tạo dự án Next.js App Router với TypeScript và thiết lập cấu trúc thư mục cơ bản.](1.1-week1/)

**Tuần 2:** [Cấu hình Axios Interceptors phục vụ việc đính kèm Request ID truy vết và tự động xử lý JWT token.](1.2-week2/)

**Tuần 3:** [Hiện thực hóa quản lý trạng thái đăng nhập toàn cục và thiết kế khung giao diện (Layout Skeletons).](1.3-week3/)

**Tuần 4:** [Xây dựng logic giỏ hàng (Cart) ở Frontend kết hợp lưu trữ Local Storage cho khách vãng lai.](1.4-week4/)

**Tuần 5:** [Phát triển hệ thống REST API giỏ hàng ở Backend bằng Spring Boot và thiết kế cơ sở dữ liệu.](1.5-week5/)

**Tuần 6:** [Hiện thực hóa cơ chế đồng bộ giỏ hàng (Cart Sync) từ client lên server khi đăng nhập thành công.](1.6-week6/)

**Tuần 7:** [Cài đặt, cấu hình và khởi tạo hạ tầng hàng đợi tin nhắn Message Queue (RabbitMQ/Kafka).](1.7-week7/)

**Tuần 8:** [Xây dựng mô hình Publisher gửi sự kiện (Email, Notification) bất đồng bộ sang Message Queue.](1.8-week8/)

**Tuần 9:** [Xây dựng bộ Consumer lắng nghe hàng đợi để xử lý gửi Email và Thông báo hệ thống.](1.9-week9/)

**Tuần 10:** [Viết mã kiểm thử Unit Test và Integration Test đảm bảo độ ổn định của toàn hệ thống Backend.](1.10-week10/)

**Tuần 11:** [Thiết kế và cấu hình quy trình tích hợp liên tục tự động (CI/CD Pipeline) bằng GitLab CI.](1.11-week11/)

**Tuần 12:** [Tự động hóa đóng gói ứng dụng Docker Image và đẩy lên GitLab Container Registry.](1.12-week12/)
